# Процедура: генерация трека (полный пайплайн)

## Шаг 0 — Ресёрч базы знаний

Перед генерацией ВСЕГДА читай релевантные файлы из KB:

1. **Жанр/BPM:** `suno/genre_bpm_map.md` — точный BPM для жанра
2. **Вокал:** `suno/vocal_anchors.md` — подобрать вокальный якорь под голос
3. **Структура/поведение:** `suno/behavior_tags.md` — behavior tags вместо тупых [Verse]/[Chorus]
4. **Промпт-правила:** `suno/prompts.md` — формат тегов, Style vs Lyrics
5. **v5.5 синтаксис:** `suno/v5-advanced.md` — compound hyphenation, triad of contrast, архетипы
6. **Жанровые гайды:** `suno/siliconsense_guides.md` — структура, аккорды, русские особенности
7. **Вокал-триггеры:** `suno/vocals.md` — что вызывает визг, что держит ровный вокал
8. **Аранжировка:** `suno/arrangement.md` — фиксы пропадающих инструментов
9. **Персона:** `suno/persona.md` — дрейф голоса, ограничения
10. **Баги:** `suno/quirks.md` — известные глюки

Не все файлы нужны каждый раз — выбирай по запросу. Но минимум: prompts.md + genre_bpm_map + vocal_anchors.

## Шаг 1 — Уточнение с юзером

Если запрос абстрактный («сделай трек», «что-нибудь крутое») — задай 2-4 вопроса:

- Жанр/стиль? (или дай 2-3 варианта на выбор)
- Вокал: мужской/женский/инструментал? Характер голоса?
- Настроение/эмоция?
- BPM или темп? (медленный/средний/быстрый)
- Есть ли текст или написать?
- Язык текста?
- Референс: артист, трек, эпоха?

Не задавай ВСЕ вопросы — выбери 2-3 ключевых. Чем точнее запрос — тем меньше вопросов.

Если запрос конкретный (жанр, стиль, текст есть) — переходи сразу к генерации.

## Шаг 2 — Сборка промпта

1. **Lyrics box:**
   - Структура через `[ ]` квадратные скобки, английский: `[Intro]`, `[Verse]`, `[Chorus]`, `[Build]`, `[Drop]`, `[Outro]`
   - Behavior tags: `[Structure: Anthemic Peak]`, `[Whispered]`, `[Ad-libs]`
   - Подпевка в `( )` круглых скобках
   - Если текст пишет Краб — пройти через анти-AI пайплайн (см. SOUL.md)
   - Если instrumental=true — структура в `[ ]` описывает секции, текст не нужен

2. **Style box:**
   - Жанр, настроение, инструменты, вокальный стиль, эпоха, продакшен
   - Vocal anchor: `[Vocal: male, deep husky timbre...]`
   - BPM числом
   - Compound hyphenation для экономии (v5.5: 120 символов лимит)
   - НИКАКИХ структурных тегов в Style

3. **Model:**
   - v5.5 (`sun/chirp-v5-5`) — лучший контроль, чище вокал
   - v5 (`sun/chirp-v5`) — быстрее, чуть сырее
   - v4.5-plus (`sun/chirp-v4-5-plus`) — длинные треки до 8 мин
   - По умолчанию: v5.5

4. **Negative tags:** исключить нежелательное (upbeat, female vocal, EDM...)

## Шаг 3 — Генерация + скачивание

1. Вызвать `chirp_generate` с собранным промптом
2. ID песни доступен сразу в ответе (`songs[].id`) → ссылка suno.com/song/{id}
3. Скачать оба варианта через `curl` с audiopipe.suno.ai
4. Проверить ID в ID3-теге MP3: `comment` = `made with suno; created=...; id={uuid}`
   - можно извлечь через python regex: `re.findall(r'id=([0-9a-f-]+)', comment)`
5. Отдать юзеру: файлы (MEDIA:) + ссылка suno.com/song/{id}
