## 27. Шаблоны генерации

Готовые каркасы с плейсхолдерами. Подставь параметры из вектора (раздел 24) и генерируй.

### 27.1 Шаблон: verse-chorus (стандарт)

```
[Intro]
{instrumental_or_whisper}

[Verse 1]
{voice.intro}                    # где герой, что делает
{image_anchor_first}             # первый образ-якорь
{concrete_detail}                # конкретика (запах/фактура/звук)
{action_with_body}               # телесное действие
{register_contrast_line}         # столкновение low/high
{internal_rhyme_line}            # строка с внутренней рифмой

[Pre-Chorus]
{short_tension_line}
{short_tension_line_2}           # сжатие, нарастание

[Chorus]
{HOOK}                           # 1–2 строки, заедает
{HOOK_repeat_or_variation}
{contrast_line}                  # контраст к хуку

[Verse 2]
{voice.continuation}
{image_anchor_repeat}            # повтор якоря с нарастанием
{new_concrete_detail}
{register_contrast_line_2}
{internal_rhyme_line_2}

[Chorus]
{HOOK}
{HOOK_repeat_or_variation}
{contrast_line}

[Bridge]
{new_angle}                      # новый ракурс, новый образ
{shift}                          # сдвиг голоса/тона

[Chorus]
{HOOK}
{HOOK_repeat}
{outro_line}                     # затухание

[Outro]
{HOOK_whisper_or_repeat}
{vocalisation_optional}          # la-la / tu-tu
```

```
{ANCHOR}
{ANCHOR}

{vocalisation}
```

### 27.3 Шаблон: рэп-куплет (Хаски/Скриптонит-стиль)

```
[Verse]
{line_1: 3-4 удара, тоника}
  {internal_rhyme_a}             # внутренняя рифма
{line_2: 3-4 удара, сленг}
  {end_rhyme_a}                  # концевая рифма (неточная)
{line_3: 2-3 удара, короче}      # удар
  {internal_rhyme_b}
{line_4: 4 удара, регистровый контраст}  # сленг + высокое
  {end_rhyme_b}
{line_5: 3 удара, телесный код}  # тело грязное/больное
  {internal_rhyme_c}
{line_6: 3-4 удара, фонетика}    # б/п/к/т — перкуссия
  {end_rhyme_c}
{line_7: 2 удара, панчлайн}      # отдельная строка-удар
{line_8: 4 удара, образ-якорь}   # повтор якоря
  {end_rhyme_d}
# 16 строк — стандарт куплета
```

### 27.4 Шаблон: spoken word / декламация (Макулатура)

## 28. Функция оценки

Формальная оценка «живости» текста. 10 параметров, каждый 0–1, сумма × 10 = score от 0 до 10.

### 28.1 Параметры

```
PARAMETERS (each 0.0–1.0):

P1 concreteness     — конкретика (объекты, тело, сенсорика) vs абстракции
P2 rhythm_variance  — вариативность ритма (разная длина, переносы) vs ровность
P3 rhyme_quality    — неточные/ассонансные/внутренние vs банальные/глагольные
P4 register_contrast— столкновение high/low vs один регистр
P5 body_code        — телесность (грязное/больное/усталое) vs романтизация
P6 ugly_aesthetics  — уродливое как персонаж vs абстрактный фон
P7 phonetics        — звукопись (аллитерация/вокализация) vs отсутствие
P8 hook_strength    — заедающий хук (1–2 строки) vs чеклист/отсутствие
P9 voice_embodiment — герой имеет тело/место/действие vs абстрактный «я»
P10 anti_ai_clean   — отсутствие AI-маркеров (детектор 25) vs их наличие
P11 singability     — пропеваемость: открытые гласные на длинных нотах и концах строк,
    нет кластеров согласных в драйвовых местах, припев поётся с первого раза
P12 cohesion        — цельность: интро/бридж/аутро перекликаются,
    образ-якорь копит смысл, песня читается как одно высказывание
```

### 28.2 Формула

```
# v1.0 (без жанровых весов):
score = (P1 + P2 + P3 + P4 + P5 + P6 + P7 + P8 + P9 + P10) / 10 × 10

# v2.0 (с жанровыми весами и P11/P12):
score = Σ(w_i × P_i) / Σ(w_i) × 10

# Упрощённо: score = sum(P1..P10)
# Диапазон: 0.0 – 10.0
```

### 28.2b Жанровые веса (v2.0)

> Источник: Notion-апдейт §25–28 v2.0. Для жанров вне таблицы — все веса 1.0 (эквивалент v1).

| Параметр | рэп | постпанк | думер | поппанк | неофолк | darksynth |
|---|---|---|---|---|---|---|
| P1 concreteness | 1.2 | 1.0 | 1.0 | 0.8 | 1.0 | 1.2 |
| P2 rhythm_variance | 1.2 | 1.0 | 0.8 | 0.3 | 1.0 | 1.0 |
| P3 rhyme_quality | 1.2 | 0.3 | 0.5 | 0.3 | 1.0 | 1.0 |
| P4 register_contrast | 1.2 | 0.5 | 0.3 | 0.5 | 0.5 | 1.0 |
| P5 body_code | 1.2 | 0.5 | 0.8 | 0.3 | 0.8 | 1.0 |
| P6 ugly_aesthetics | 1.2 | 1.0 | 1.2 | 0.3 | 0.5 | 0.8 |
| P7 phonetics | 1.2 | 1.0 | 0.5 | 0.5 | 0.8 | 1.0 |
| P8 hook_strength | 1.0 | 1.2 | 0.8 | 1.5 | 1.0 | 1.2 |
| P9 voice_embodiment | 1.0 | 1.0 | 1.2 | 0.8 | 1.0 | 1.0 |
| P10 anti_ai_clean | 1.5 | 1.5 | 1.5 | 1.5 | 1.5 | 1.5 |
| P11 singability | 0.8 | 0.8 | 0.8 | 1.2 | 1.0 | 1.5 |
| P12 cohesion | 1.0 | 1.2 | 0.8 | 0.8 | 1.0 | 1.2 |

- В поппанке низкий вес P3 означает: банальная AABB — осознанный приём, оцениваем не «неточность», а осознанность (§12.4).
- Для жанров вне таблицы — все веса 1.0 (эквивалент v1).

### 28.3 Оценка параметров (рубрикатор)

**P1 concreteness:**
- 1.0 — все образы конкретные (объекты, тело, запах), 0 абстракций
- 0.5 — смесь конкретного и абстрактного
- 0.0 — преимущественно абстракции

**P2 rhythm_variance:**
- 1.0 — разная длина строк, есть переносы, есть удар (короткая после длинной)
- 0.5 — некоторая вариативность
- 0.0 — всё ровно (исключение: поппанк/мантра — тогда 1.0 если осознанно)

**P3 rhyme_quality:**
- 1.0 — неточные/ассонансные/внутренние, нет глагольных, нет банальных
- 0.5 — смесь, есть 1–2 проходные
- 0.0 — глагольные/банальные преобладают

**P4 register_contrast:**
- 1.0 — явное столкновение high/low в строфе
- 0.5 — контраст есть, но слабый
- 0.0 — один регистр (исключение: ploho/Shortparis — 1.0 если осознанно)

**P5 body_code:**
- 1.0 — тело грязное/больное/усталое, физическое действие = эмоция
- 0.5 — тело есть, но романтизировано
- 0.0 — тело отсутствует или идеализировано

**P6 ugly_aesthetics:**
- 1.0 — уродливое как персонаж (панелька, подворотня — герой)
- 0.5 — уродливое как фон
- 0.0 — нет уродливого или абстрактный фон (исключение: неофолк — 0.5–1.0)

**P7 phonetics:**
- 1.0 — аллитерация/ассонанс в ключевых строках, звуки несут ритм
- 0.5 — фонетика присутствует, но не структурно
- 0.0 — звукопись отсутствует

**P8 hook_strength:**
- 1.0 — заедающий хук 1–2 строки, в припеве/начале/конце
- 0.5 — хук есть, но слабый/длинный
- 0.0 — хука нет или припев-чеклист

**P9 voice_embodiment:**
- 1.0 — герой имеет тело, место, действие, не оправдывается
- 0.5 — герой частично телесен
- 0.0 — абстрактный «я» без тела, объясняет позицию

**P10 anti_ai_clean:**
- 1.0 — ни одного AI-маркера (детектор 25 = 0 флагов)
- 0.5 — 1–2 флага низкого веса
- 0.0 — 3+ флага или любой флаг веса ≥ 1.5

### 28.4 Вердикт

```
score ≥ 8.0  → EXCELLENT (живое, готово)
score 7.0–7.9 → GOOD (годится, можно улучшить)
score 5.0–6.9 → SUSPECT (нужен REPAIR по флагам)
score < 5.0  → SLOP (полный REPAIR)
```

**Порог прохода: ≥ 7.0.**

### 28.4b Hard-fail поверх score (v2.0)

- Любой флаг веса 2.0 → вердикт максимум SUSPECT (cap 6.9), даже при высоком score.

### 28.4c REPAIR-приоритизация (v2.0)

```
priority(fix) = вес_флага × лёгкость_правки
  лёгкость: 1.0 = точечная замена слова
            0.5 = переписать строку
            0.25 = переписать строфу
Чинить сверху вниз по priority. Максимум 3 итерации (без изменений).
```

### 28.4d Формат delta-отчёта (для AUDIT/REPAIR, v2.0)

```
score: X.X (жанр: {genre}, веса: v2)
hard-fail: {да/нет, какой флаг}
top-3 delta: P{n} → {что делать, ссылка на §26.x}
```

### 28.5 Пример оценки

Текст (слоуп, «серверная» первая версия):
```
слёзы в формате WAV       → P10: 0 (tech_metaphor, вес 2.0)
крик — в индекс           → P10: 0 (tech_metaphor)
вызов — в лог             → P10: 0
помню всё                 → P10: 0 (position_explanation)
я не сужу                 → P10: 0 (position_explanation)
припев = чеклист          → P8: 0
одинаковая длина строк    → P2: 0
```
Score: P1=0.3, P2=0, P3=0.5, P4=0, P5=0, P6=0.2, P7=0, P8=0, P9=0.3, P10=0 → **1.3/10 = SLOP**

Текст (живое, «серверная» последняя версия):
```
серверная не спит ×2      → P8: 1 (рефрен-якорь)
ты спишь / я смотрю       → P9: 0.8 (герой, действие)
твой стук повтор          → P2: 0.7 (ритм)
ты вошла в три четырнадцать → P1: 0.8 (конкретика)
но: нет регистрового контраста → P4: 0
нет эстетики уродливого    → P6: 0.2
нет фонетики               → P7: 0
нет мата/тела              → P5: 0
```
Score: P1=0.8, P2=0.7, P3=0.7, P4=0, P5=0, P6=0.2, P7=0, P8=1, P9=0.8, P10=0.7 → **4.9/10 = SUSPECT**

Что улучшить (delta): P4 (добавить регистровый контраст), P5/P6 (тело, уродливое), P7 (фонетика). После REPAIR → ~7.5.

### 28.6 Цикл REPAIR

```
WHILE score < 7.0:
  1. Прогнать детектор (25) → найти флаги
  2. Применить трансформации (26) по флагам
  3. Оценить заново (28)
  4. IF score не растёт 2 итерации → STOP, отдать как есть с пометкой
```

Максимум 3 итерации. Если после 3 не достигнут 7.0 — текст требует ручного вмешательства, отдать с delta-отчётом.
