## 26. Правила трансформации

Каждое правило: `X → Y`. Вход — сработавший патрон из раздела 25.

### 26.1 marker_word → concrete_image

```
RULE: абстрактное слово-маркер → конкретный образ (объект + запах + время + действие)
PRINCIPLE: один специфический запах/тактильность сильнее пяти абстракций

EXAMPLES:
  "вечность"        → "запах мокрого асфальта в три ночи"
  "бесконечность"   → "дорога, что не кончается за окном"
  "симфония"        → "гул труб в подвале, где коты"
  "гармония"        → "тишина после того, как соседи заткнулись"
  "мелодия души"    → "свист чайника на пустой кухне"
  "навсегда"        → "пока не остынет чай"
  "волшебный"       → "тот, от которого мурашки" (или выкинуть)
  "глубокий"(filler) → выкинуть

TRANSFORM ALGORITHM:
  1. Найди ядро смысла (что хотел сказать)
  2. Найди конкретный объект/действие, несущий тот же смысл
  3. Добавь сенсорику (запах/звук/тактильность)
  4. Проверь: образ бьёт, не объясняется после
```

### 26.2 tech_metaphor → bodily_or_sensory

```
RULE: тех-метафора эмоции → телесное действие или сенсорика
EXCEPTION: тех-лексика как грунт (контекст = технический мир героя) → оставить, не метафора

EXAMPLES:
  "слёзы в формате WAV"        → "слёзы капают на клавиатуру"
  "сердце в архиве"            → "сердце стучит в рёбра, как в закрытый ящик"
  "душа в облаке"              → "душа — комом в горле"
  "любовь в кэше"              → "помню твои руки, тёплые"
  "память в логах"             → "помню, как ты смеялся у окна"
  "перезагрузка чувств"        → "проснулся, и снова с начала"
  "код души"                   → выкинуть, заменить действием
```

### 26.3 organ_cliche → body_action

```
RULE: штамп про сердце/душу → физическое действие, несущее эмоцию
PRINCIPLE: эмоция подаётся через тело, не называется

EXAMPLES:
  "сердце бьётся"      → "рука дрожит на столе"
  "сердце колотится"   → "не хватает дыхания на лестнице"
  "сердце рвётся"      → "тяжесть в груди, не встать"
  "душа рвётся"        → "хочется крикнуть, но молчу"
  "душа кричит"        → "голос сел, шепчу"
  "душа горит"         → "жарко, хотя окна настежь"
```

### 26.4 binary_light_dark → concrete_contrast

```
RULE: рамка свет/тьма → конкретный контраст (через объекты, не абстракции)
PRINCIPLE: контраст через конкретное, не через свет/тьму

EXAMPLES:
  "во тьме" + "свет внутри" → "на улице дождь / в комнате — лампа на столе"
  "между светом и тьмой"    → "между окном и дверью"
  "тьма поглощает"          → "облака затянули всё"
  "свет надежды"            → "луч из щели в шторе"

ALGORITHM:
  1. Найди два полюса (что противопоставляется)
  2. Найди конкретные объекты для каждого полюса
  3. Замени свет/тьму на эти объекты
  4. Убедись, что контраст остался, а абстракция ушла
```

### 26.5 hypophora → remove_question_or_merge

```
RULE: вопрос-ответ → либо убрать вопрос, либо объединить в одно утверждение
PRINCIPLE: не задавай вопрос, чтобы самому на него ответить

EXAMPLES:
  BAD:  "Что мне осталось? Только память."
  FIX1: "Только память."  (убрать вопрос)
  FIX2: "Осталась память — и больше ничего."  (слить)
  FIX3: "Память. И больше ничего."  (парцелляция)

ALGORITHM:
  IF ответ = прямое содержимое вопроса → убрать вопрос, оставить ответ
  ELSE → слить вопрос и ответ в одно утверждение
```

### 26.6 not_x_but_y → collapse_to_single_statement

```
RULE: «не просто X — это Y» / «не X, а Y» → одно утверждение
PRINCIPLE: не противопоставляй ради противопоставления

EXAMPLES:
  BAD:  "Это было не просто любовь — это было проклятие."
  FIX1: "Это было проклятие."  (оставить сильное)
  FIX2: "Любовь обернулась проклятием."  (слить через глагол)
  FIX3: "Проклятие. Не любовь."  (парцелляция, reverse)

ALGORITHM:
  1. Определи, что сильнее — X или Y
  2. Оставь сильное, выкинь слабое
  3. Или слей через глагол, если оба нужны
```

### 26.7 noun_stack → break_with_verbs

```
RULE: 3+ существительных подряд → разбить глаголами
PRINCIPLE: глагол — двигатель, существительное — якорь

EXAMPLES:
  BAD:  "одиночество пустота тишина ночи"
  FIX1: "одиночество. пустота. тишина. и ночь идёт."
  FIX2: "одиночество приходит, пустота остаётся, тишина ночи — последней"
  FIX3: "ночью — одиночество, пустота, тишина. и ничего не происходит."

ALGORITHM:
  1. Вставь глагол между 2-м и 3-м существительным
  2. Или сделай парцелляцию (каждое — отдельная строка/фраза)
  3. Или замени 2-е существительное на глагол
```

### 26.8 adj_pile → keep_max_two

```
RULE: 3+ прилагательных перед существительным → оставить максимум 2, третье только если бьёт
PRINCIPLE: нанизывание = слоуп

EXAMPLES:
  BAD:  "холодный пустой мёртвый серый дом"
  FIX1: "холодный мёртвый дом"  (оставить 2 бьющих)
  FIX2: "серый дом, холодный и пустой"  (разнести)
  FIX3: "дом. холодный. мёртвый."  (парцелляция)
```

### 26.9 verb_rhyme / banal_rhyme → loose_or_assonance

```
RULE: глагольная/банальная рифма → неточная / ассонансная / смысловая
PRINCIPLE: один точный удар лучше двух проходных

EXAMPLES:
  BAD:  "любить — забыть"  → FIX: "любить — губить"  (смысловая, не банальная)
  BAD:  "идти — найти"     → FIX: "идти — по пути"  (ассонансная, предлог)
  BAD:  "кровь — любовь"   → FIX: "кровь — морковь"  (если гротеск) / "кровь — вновь"  (ассонанс)
  BAD:  "ночь — дочь"      → FIX: "ночь — прочь" → уже банальная → "ночь — мочь"  (ассонанс, глагол-модальный)
  BAD:  "слёзы — грёзы"    → FIX: "слёзы — берёзы"  (ассонанс, конкретный образ)

ALGORITHM:
  1. Определи тип рифмы (глагольная / банальная пара)
  2. Найди слово с тем же ударным гласным, но:
     - существительное (не глагол)
     - не из банального списка
     - несёт смысл
  3. Если нет — ассонанс (совпадение гласных, согласные разные)
  4. Если нет — смысловая рифма (рифмуются смыслы, не звуки)
```

### 26.10 chorus_checklist → collapse_to_hook

```
RULE: припев-чеклист → припев-крюк (одна бьющая фраза, повторённая с нарастанием)
PRINCIPLE: припев — не статус-борд, а удар

EXAMPLES:
  BAD (чеклист):
    "температура — норма
     влажность — допуск
     давление — в пределах
     сердце — работает"
  FIX (крюк):
    "серверная не спит
     серверная не спит
     ты спишь
     а я смотрю"

ALGORITHM:
  1. Найди общее в строках чеклиста (что они все описывают)
  2. Сформулируй это как одну бьющую фразу
  3. Повтори 2–3 раза с нарастанием
  4. Добавь контрастную строку (как «ты спишь»)
```

### 26.11 position_explanation → show_through_action

```
RULE: «я не сужу / помню всё / не мне судить» → убрать, показать через действие
PRINCIPLE: герой не оправдывается, не объясняет позицию

EXAMPLES:
  BAD:  "я не сужу его"     → FIX: "он пьёт. я наливаю ему."  (действие)
  BAD:  "я помню всё"       → FIX: "помню запах твоих волос"  (конкретика)
  BAD:  "не мне судить"     → FIX: выкинуть, пусть действие говорит
  BAD:  "я не берусь объяснить" → FIX: выкинуть, просто показать

ALGORITHM:
  1. Найди, что объясняется (позиция героя)
  2. Найди действие/образ, который показывает то же без слов
  3. Замени объяснение на действие
```

### 26.12 truncation → finish_to_end

```
RULE: «[Продолжение следует] / Потом было…» → дописать до конца
PRINCIPLE: без усечения, никогда

ALGORITHM:
  1. Определи, где обрезано
  2. Допиши финал — образ, действие, открытая развязка
  3. Не закрывай моралью («и тогда я понял»)
  4. Заканчивай действием или открытой развязкой, ведущей в следующую сцену
```

### 26.13 uniform_line_length → vary_one_line

```
RULE: одинаковая длина строк → варьировать (если не поппанк и не мантра-рефрен)
PRINCIPLE: разная длина = живой ритм, одинаковая = машина

ALGORITHM:
  1. Найди самую сильную строку в строфе
  2. Сделай её короче (удар)
  3. Или сделай одну строку длиннее (разлив)
  4. Цель: variance > 1.5 слога
```

### 26.14 parallel_no_shift → add_progression_or_break

```
RULE: параллелизм без нарастания → нарастание (градация) или слом (сдвиг)
PRINCIPLE: повтор должен копить, не повторяться впустую

EXAMPLES:
  BAD:  "я помню / я знаю / я вижу / я чувствую" (×4 ровно)
  FIX1 (градация): "я помню / я знаю / я вижу / я не могу забыть"  (нарастание)
  FIX2 (слом):     "я помню / я знаю / я вижу / а потом — пусто"  (сдвиг)
  FIX3 (сокращение): "помню / знаю / вижу / забыл"  (нарастание через укорачивание)

ALGORITHM:
  1. Найди, что повторяется (слово / структура)
  2. Добавь нарастание: каждое следующее сильнее/длиннее/конкретнее
  3. Или сломай последнюю строку — сдвиг, контраст
  4. Или сократи — нарастание через укорачивание
```

### 26.16 kantselyarit → spoken_register

```
"является частью меня"  → "это во мне"
"данный город"          → "этот город"
"осуществляю движение"  → "иду"
ALGORITHM: скажи фразу вслух соседу — как сказал бы человек, так и пиши
```

### 26.17 genitive_metaphor → concrete_object

```
"осколки памяти" → "фотка с обгоревшим углом"
"тени прошлого"  → "твоя куртка всё ещё на крюке"
"эхо души"       → "гул в подъезде после хлопка двери"
"нити судьбы"    → "провода над двором, провисшие к пятому этажу"
ALGORITHM:
  1. Найди, ЧТО помнится/тянется/болит — конкретный предмет
  2. Дай предмету действие или след (обгорел, висит, гудит)
  3. Проверь: образ можно сфотографировать? Да → ок
```

### 26.18 em_dash_cascade → one_insertion_max

```
BAD: "она открыла книгу — страницы пожелтели — и нашла карту — старую"
FIX: "она открыла книгу. страницы жёлтые. внутри — карта"
ALGORITHM: оставь одно тире (самое сильное), остальное — точки/парцелляция
```

### 26.19 simile_chain → single_or_action

```
BAD:  "ты словно ветер, будто дождь, как гроза"
FIX1: "ты — сквозняк: дверь хлопнула, и нет"      (одно, конкретное)
FIX2: "ты хлопаешь дверью, как ставней в грозу"   (сравнение через действие)
```

### 26.20 sentiment_flatline → dirt_and_body

```
ALGORITHM:
  1. Найди самое «чистое» место текста
  2. Добавь одну грязную/резкую деталь из мира героя (§9, §14)
  3. Проверь тон: хотя бы одна строка должна царапать
```

### 26.21 perfect_grammar → break_syntax

```
BAD: "Я смотрю на город, и я вспоминаю тебя, и мне становится грустно"
FIX: "смотрю на город. вспоминаю. ну и пусть"
ALGORITHM: убери союзы, режь на куски, разрешай безглагольные строки
```

### 26.22 school_arc → in_medias_res + open_ending

```
ALGORITHM:
  1. Выкини куплет-экспозицию — начни с середины сцены
  2. Финал: действие или открытая развязка, не вывод
BAD: "и тогда я понял, что жизнь одна"
FIX: "автобус ушёл. я остался стоять"
```

### 26.23 triple_rhetoric → keep_one

```
BAD: "боль, страх и пустота"
FIX: "страх" → потом конкретизируй: "страх, что телефон молчит"
```

### 26.24 genre_autopilot → reframe_or_ground

```
"неон"         → конкретный источник света: "вывеска ломбарда мигает через штору"
"город спит"   → "окна гаснут по стояку, наше — последнее"
"по венам ток" → телесное: "пальцы дёргает, как от оголённого провода"
ALGORITHM: жанровый штамп либо переворачивается (ирония/сдвиг),
  либо заземляется конкретным объектом
```

### 26.25 vague_deixis → concrete_referent

```
BAD: "всё это было не зря"
FIX: "эти три зимы были не зря"  (или вырезать строку)
```
