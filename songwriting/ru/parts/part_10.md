## 21. Упражнения

### 21.1 Ритм

1. Возьми любое стихотворение и пересчитай удары в каждой строке. Сравни с соседними.
2. Напиши 4 строки тоническим стихом, 3 удара на строку, разная длина.
3. Напиши 4 строки, где одна строка короткая после длинных (удар).

### 21.2 Рифма

1. Возьми глагольную рифму (любить—забыть) и замени на неточную / ассонансную.
2. Напиши строку с внутренней рифмой (не концевой).
3. Найди 5 неточных рифм на слово «окно».

### 21.3 Образ

1. Замени абстракцию («одиночество») на конкретный образ (объект + запах + время).
2. Напиши образ-якорь и повтори его 3 раза в строфе с нарастанием.
3. Напиши оксюморон («харчок золота» — уродливое в красивом).

### 21.4 Регистровый контраст

1. Напиши строку, где сленг стоит рядом с архаизмом / классикой.
2. Напиши строфу, где одна строка — быт, другая — философия.

### 21.5 Голос

1. Опиши лирического героя: где он, что делает руками, куда смотрит, какой регистр.
2. Напиши 4 строки от лица этого героя, без объяснения позиции.

### 21.6 Хук

1. Напиши фразовый хук (1 строка, заедает).
2. Напиши ритмический хук (2 строки, контрастный ритм).
3. Напиши припев-крюк (не чеклист).

### 21.7 Анти-AI

1. Возьми AI-сгенерированный текст и найди 5 маркеров (раздел 16).
2. Замени каждый маркер конкретным образом.
3. Прочитай результат вслух — где спотыкаешься, там сильное место.

### 21.8 Калибровка

1. Выбери автора из пантеона. Напиши 4 строки в его стиле, по рецепту (раздел 18).
2. Прочитай вслух. Похоже? Что не так? Исправь.
3. Пройди анти-AI-чек-лист.

### 21.9 Полный цикл

1. Выбери тему (например, «человек в серверной ночью»).
2. Найди голос героя (раздел 7).
3. Найди образ-якорь (раздел 5).
4. Определи форму (раздел 4).
5. Найди хук (раздел 13).
6. Напиши куплет, припев, bridge.
7. Прочитай вслух, считай удары, слушай фонетику.
8. Третий проход — нож: режь лишнее.
9. Пройди пайплайн (17.1) как финальную проверку.
10. Пройди сводный чек-лист (20.2).

## 22. AI-операционная карта

Этот блок переводит энциклопедию из «справочника для чтения» в **исполняемый документ** — набор алгоритмов, которые AI-агент выполняет пошагово. Если разделы 1–21 — теория, то 22–30 — операционная система.

### 22.1 Что значит «AI-friendly»

Документ оптимизирован для потребления языковой моделью, а не только человеком. Это значит:

1. **Явные ветвления** — вместо «подумай, какой тон уместен» → `IF жанр=постпанк THEN тон=мёртвый ELSE IF жанр=рэп THEN тон=плотный`
2. **Векторы вместо прозы** — стиль автора = числовые/категориальные параметры, не абзац описания
3. **Патроны детекции** — regex-подобные правила для поиска AI-маркеров
4. **Правила трансформации** — `X → Y` (что заменить, на что)
5. **Шаблоны (templates)** — готовые каркасы с плейсхолдерами
6. **Функция оценки** — формальная оценка «живости» текста
7. **Промпт-скаффолды** — системные промпты для генерации в нужном стиле

### 22.2 Режимы работы с энциклопедией

| Режим | Вход | Что использует | Выход |
|---|---|---|---|
| **GENERATE** | тема/тема+автор | 23 (деревья), 24 (векторы), 27 (шаблоны), 29 (скаффолды) | новый текст |
| **REPAIR** | существующий текст | 25 (детекция), 26 (трансформация), 28 (оценка) | исправленный текст |
| **CALIBRATE** | текст + целевой автор | 24 (векторы), 28 (оценка) | delta-отчёт (что не так) |
| **AUDIT** | текст | 25 (детекция), 28 (оценка) | verdict + score |

### 22.3 Порядок исполнения (канонический)

```
GENERATE:  23 (дерево: жанр→голос→форма) → 24 (вектор стиля) → 27 (шаблон) → 17 (пайплайн) → 28 (оценка) → 26 (если score<7)
REPAIR:    25 (детекция маркеров) → 26 (трансформация) → 28 (оценка) → цикл до score≥7
CALIBRATE: 24 (вектор целевого автора) → 28 (оценка текста по вектору) → 26 (трансформация дельт)
AUDIT:     25 (детекция) → 28 (оценка) → verdict
```

Порог прохода: **score ≥ 7/10** по функции оценки (раздел 28). Ниже — цикл REPAIR.

## 23. Деревья решений

Явные ветвления. Исполняй сверху вниз.

### 23.1 Дерево: выбор голоса лирического героя

```
INPUT: тема, жанр
  │
  ├─ жанр = рэп?
  │   ├─ тема = быт/подворотня/панелька → ГОЛОС = маргинал_на_дне (Хаски)
  │   ├─ тема = социальное наблюдение → ГОЛОС = свидетель_без_позиции (Скриптонит)
  │   ├─ тема = литература/философия → ГОЛОС = интеллектуал_на_дне (Pyrokinesis)
  │   ├─ тема = нуар/город/история → ГОЛОС = нарратор_нуар (ЛСП)
  │   ├─ тема = поколение/исповедь → ГОЛОС = исповедник (25/17)
  │   └─ тема = абсурд/насилие → ГОЛОС = маска_абсурда (Кровосток)
  │
  ├─ жанр = постпанк?
  │   ├─ тема = апатия/быт → ГОЛОС = холодный_декламатор (ploho)
  │   ├─ тема = тревога/страх → ГОЛОС = интеллектуал_на_дне (Shortparis)
  │   └─ тема = абсурд/ирония → ГОЛОС = маска_абсурда (Буерак)
  │
  ├─ жанр = поппанк?
  │   └─ → ГОЛОС = маска_абсурда (Пошлая Молли)
  │
  ├─ жанр = инди/неофолк?
  │   └─ → ГОЛОС = меланхолик (Аффинаж/almgamba)
  │
  └─ жанр = spoken_word?
      └─ → ГОЛОС = холодный_декламатор (Макулатура)
```

### 23.2 Дерево: выбор метрики

```
INPUT: жанр
  │
  ├─ рэп → тоника / акцентный стих (3–4 удара/строка, слоги свободны)
  ├─ постпанк → верлибр (свободный стих, рифма не обязательна)
  ├─ думер → простая силлабо-тоника с разреженностью (короткие строки, воздух)
  ├─ поппанк → ровная силлабо-тоника (AABB, одинаковая длина строк — осознанно)
  ├─ инди/неофолк → дольник (1–2 слога между ударами, волна)
  └─ spoken_word → декламация, ритм = синтаксис
```

### 23.3 Дерево: выбор формы песни

```
INPUT: жанр, цель
  │
  ├─ цель = хит/вирусность → verse-chorus (ABABCB или ABACB)
  ├─ жанр = постпанк AND цель = тревога → циклическая (рефрен-якорь)
  ├─ цель = концептуальность → сквозная (through-composed)
  ├─ жанр = шансон/фолк → куплетная (строфическая)
  └─ цель = джазовая плотность → AABA
```

### 23.4 Дерево: выбор хука

```
INPUT: что заедает в материале
  │
  ├─ есть заедающая фраза → фразовый хук (1 строка)
  ├─ есть заедающий контраст → контрастный хук (2 строки)
  ├─ есть заедающий образ → образный хук (1 строка с образом)
  ├─ ритм важнее слов → ритмический хук (2 строки, контрастный ритм)
  ├─ слов не хватает → вокализация (la-la, tu-tu)
  └─ ничего не заедает → ВЕРНУТЬСЯ к материалу, хук не из воздуха
```

### 23.5 Дерево: допустимость мата

```
INPUT: голос, тема
  │
  ├─ голос = маргинал_на_дне AND мат несёт смысл → ДОПУСТИМ (как регистр)
  ├─ голос = маска_абсурда AND мат несёт контраст → ДОПУСТИМ
  ├─ голос = свидетель_без_позиции → осторожно, точечно
  ├─ голос = меланхолик/исповедник → почти никогда
  └─ мат ради мата / для эпатажа → ЗАПРЕЩЁН
```

### 23.6 Дерево: интертекстуальность

```
INPUT: есть ли второй слой
  │
  ├─ отсылка работает на смысл трека → включить, точечно (1/строфа)
  ├─ отсылка ради показухи → выкинуть
  ├─ религия/миф → опустить в быт, снять сакральность (Хаски-паттерн)
  └─ прямая цитата → только если она расширяет, не подменяет
```
